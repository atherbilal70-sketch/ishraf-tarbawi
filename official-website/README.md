# الموقع الرسمي لمديرية الإشراف التربوي — وزارة التربية العراقية

موقع حكومي رسمي (المرحلة الأولى: واجهة أمامية كاملة) لمديرية الإشراف التربوي، بتصميم عربي RTL حديث ومتجاوب.

> **ملاحظة:** جميع الأرقام والأخبار والشعارات في هذه النسخة **بيانات تجريبية (Placeholder)** يجب استبدالها بالمحتوى والشعارات الرسمية قبل الإطلاق.

---

## 1) المقترح الأولي للهيكلية التقنية

### المرحلة الأولى — الواجهة الأمامية (منجزة في هذا المجلد)

| المكوّن | التقنية | السبب |
|---|---|---|
| بنية الصفحات | HTML5 دلالي (Semantic) | توافق مع محركات البحث ومتطلبات الوصولية للمواقع الحكومية |
| التنسيق | TailwindCSS (عبر CDN حالياً) + `assets/css/site.css` | سرعة بناء، ألوان مخصصة للهوية، دعم RTL عبر الخصائص المنطقية `ms-/me-/ps-/pe-` |
| التفاعل | JavaScript (Vanilla) بدون مكتبات | لا حاجة لإطار عمل في هذه المرحلة؛ حجم أخف وأداء أسرع |
| الخط | Tajawal (Google Fonts) | خط عربي رسمي واضح على الشاشات |

**عند الانتقال للإنتاج:** يُستبدل Tailwind CDN ببناء محلي (Tailwind CLI أو PostCSS) لتوليد ملف CSS مضغوط ثابت لا يعتمد على اتصال خارجي.

### المرحلة الثانية — الواجهة الخلفية (Backend) المقترحة

| المكوّن | الخيار الموصى به | البديل |
|---|---|---|
| إطار العمل | **Laravel 11 (PHP)** — شائع في الاستضافات الحكومية، يوفر جاهزاً: التحقق من المدخلات، CSRF، صلاحيات، رفع ملفات آمن | Node.js (NestJS) أو Django |
| قاعدة البيانات | **PostgreSQL** | MySQL / MariaDB |
| تخزين المرفقات (الهويات) | قرص خاص **خارج مجلد الويب العام** (Laravel private disk) — لا يمكن الوصول للملف إلا عبر رابط موقّع مؤقت لموظف مخوّل | S3-متوافق داخلي (MinIO) |
| الإشعارات | بريد إلكتروني (SMTP الوزارة) + إمكانية SMS لاحقاً | — |
| لوحة إدارة الشكاوى | Laravel + Filament (لوحة عربية RTL جاهزة) | لوحة مخصصة |

### واجهات API المقترحة للشكاوى

```
POST   /api/complaints            → استلام الشكوى + المرفق، توليد رقم التتبع، إرسال إشعار
GET    /api/complaints/{ref}      → الاستعلام عن حالة الشكوى برقم التتبع (بيانات محدودة فقط)
GET    /api/admin/complaints      → (محمي) قائمة الشكاوى للموظفين مع فلترة
PATCH  /api/admin/complaints/{id} → (محمي) تحديث الحالة (قيد المراجعة/قيد المعالجة/منجزة/مرفوضة)
```

### الأمان (إلزامي قبل الإطلاق)

- **HTTPS** حصراً مع HSTS.
- المرفقات: فحص النوع الحقيقي للملف (MIME sniffing) وليس الامتداد فقط، حد أقصى 5MB، إعادة تسمية عشوائية، تخزين خارج `public/`، وفحص فيروسات (ClamAV).
- **Rate Limiting** على استمارة الشكاوى لمنع الإغراق، مع CAPTCHA (مثل hCaptcha أو Cloudflare Turnstile).
- تشفير البيانات الحساسة في قاعدة البيانات، ونسخ احتياطي يومي مشفّر.
- سجلّ تدقيق (Audit Log) لكل اطّلاع موظف على مرفقات الهويات.
- التحقق من كل المدخلات في الخادم (التحقق في المتصفح تجربةُ استخدام فقط، لا حماية).

## 2) محتوى هذا المجلد (المرحلة الأولى)

```
official-website/
├── index.html              الصفحة الرئيسية (سلايدر، عن المديرية، إحصائيات،
│                           الإنجازات والأنشطة مع فلترة وتكبير، جدول النتائج مع بحث وفلترة)
├── complaints.html         بوابة الشكاوى الإلكترونية + الاستعلام برقم التتبع
├── assets/
│   ├── css/site.css        تنسيقات مخصصة مكمّلة لـ Tailwind
│   └── js/
│       ├── main.js         القائمة، السلايدر، العدادات، فلترة الأخبار والنتائج
│       └── complaints.js   التحقق من الاستمارة، رفع الملفات، توليد رقم التتبع
└── README.md
```

## 3) التشغيل محلياً

لا يحتاج الموقع في هذه المرحلة إلى خادم — افتح `index.html` مباشرة في المتصفح، أو شغّل خادماً بسيطاً:

```bash
cd official-website
python3 -m http.server 8080
# ثم افتح http://localhost:8080
```

> رقم التتبع في هذه النسخة يُولَّد ويُحفَظ **محلياً في متصفح المستخدم (localStorage) للعرض التجريبي فقط**؛ في المرحلة الثانية يتولى الخادم توليده وحفظه وإرسال إشعار التأكيد بالبريد.
